# 星梦笔记轻量功能路线

这份文档记录适合继续增加、但不容易让博客变臃肿的功能。默认原则是：优先提升阅读、写作、审核和部署稳定性，暂缓会明显增加运营压力、实时能力或复杂权限边界的功能。

## 选择原则

- 每轮只新增一个核心概念，避免多个功能互相牵连。
- 优先使用浏览器原生能力和现有依赖，不轻易引入新包。
- 功能要能独立移除，不把项目绑定到单一外部服务。
- 优先服务作者工作流、读者阅读体验和后台安全。
- 在真实流量稳定前，暂缓私信、支付、复杂推荐、实时排行等重运营功能。

## 高价值低负担候选

| 功能 | 价值 | 复杂度 | 说明 |
| --- | --- | --- | --- |
| 文章阅读进度与标题锚点 | 改善长文阅读和分享定位 | 低 | 已完成：文章详情页支持进度、目录和锚点定位。 |
| 编辑器大纲面板 | 写作时快速检查文章结构 | 低 | 已完成：支持 H1-H3、行号和点击定位。 |
| 草稿版本历史 | 避免误覆盖、误删草稿 | 中 | 已完成：保存草稿时自动保留最近 5 个快照，并可在编辑器恢复。 |
| 后台举报筛选 | 提升审核效率 | 低 | 已完成：支持状态筛选、当前结果全选和批量处理。 |
| 轻量 SEO 元信息 | 改善搜索和分享预览 | 低 | 已完成：路由默认 meta、文章级标题/摘要/封面、分享链接统一。 |
| 图片替代文本 | 提升可访问性和搜索质量 | 低 | 已完成：编辑器图片资产可填写 alt，详情页和画廊会按图片自身说明渲染。 |
| 文章系列 | 组织连载或专题内容 | 中 | 文章可选归属一个系列，不必做复杂专栏系统。 |
| 私密预览链接 | 发布前给朋友或管理员预览 | 中 | 用短期 token 访问未发布内容。 |
| 后台 API 状态细化 | 便于判断线上 API 和数据库状态 | 低 | 已完成：健康卡片显示最近检查时间、响应耗时、重试状态、HTTP 状态和错误信息。 |
| JSON 导入导出备份 | 降低迁移和调试风险 | 中 | 已完成：后台支持导出全量 JSON 备份，并可在确认后导入恢复。 |

## 暂缓功能

| 功能 | 暂缓原因 | 更轻的替代方案 |
| --- | --- | --- |
| 实时通知 | 需要 WebSocket 或严格轮询策略 | 先保留本地通知中心。 |
| 完整推荐系统 | 会扩大数据模型和排序逻辑 | 继续使用标签相关文章。 |
| 站内私信 | 审核和滥用风险会快速上升 | 先加强评论和个人主页。 |
| 付费或会员 | 涉及合规、支付、退款和权限 | 先做创作者徽章。 |
| 多站点/多租户 | 会改变几乎所有权限边界 | 先保持单站点后台。 |

## 建议的后三轮

1. 增加文章系列字段，用于连载和专题阅读路径。
2. 增加私密预览链接，用于发布前给朋友或管理员预览。
3. 增加备份校验摘要，用于导入前预览用户、文章、评论等数量。

## 发布检查

- `npm run build` 必须通过。
- 新 UI 需要检查桌面和移动宽度。
- 不新增依赖，除非现有技术栈无法干净完成。
- Mock API 和真实 API 的行为要保持一致。
- GitHub Pages 部署保持绿色。
